Introducción

Este documento tiene como objetivo explicarte cómo vamos a trabajar durante el curso, qué se espera de ti y cuál es el enfoque profesional del proyecto.

No es una introducción teórica, sino una guía para entender cómo se trabaja en un entorno profesional real.

Un proyecto único, no asignaturas sueltas

Durante el curso no trabajarás en ejercicios aislados. Todas las actividades girarán en torno a un único proyecto profesional real:

  • Diseño de una base de datos
  • Implementación en SQL
  • Uso desde una aplicación web

Este enfoque reproduce cómo se trabaja en el mundo profesional, donde todo forma parte de un mismo sistema.

Unidades formativas y su papel

  • UF2175 – Diseño de bases de datos relacionales: análisis de requisitos, modelos entidad–relación y decisiones de diseño.
  • UF2176 – Definición y manipulación de datos: creación física de la base de datos, SQL y consultas reales.
  • UF2177 – Desarrollo de programas: uso de la base de datos desde una aplicación web.
Las unidades se imparten de forma coordinada y no como bloques aislados.

Cómo vamos a trabajar en el aula

El trabajo será principalmente práctico y progresivo. Cada decisión que tomes tendrá consecuencias en las siguientes fases del proyecto.

  • Trabajo individual y en grupo
  • Revisión de trabajos propios y ajenos
  • Justificación técnica de decisiones
Diseños incorrectos generan problemas posteriores, y eso forma parte del aprendizaje.

Uso de Inteligencia Artificial

Durante el curso se podrán utilizar herramientas de IA como apoyo técnico.

  • Análisis de requisitos
  • Propuestas de modelos de datos
  • Generación de SQL o código
  • Apoyo a la documentación
No se evalúa lo que genera la IA, sino tu capacidad para entenderlo, revisarlo y justificarlo.

Qué se va a evaluar

  • Análisis de problemas reales
  • Calidad del diseño de la base de datos
  • Corrección del SQL
  • Capacidad de detección de errores
  • Documentación técnica
Copiar soluciones sin entenderlas no se considera aprendizaje válido.

Por qué este proyecto es importante

El proyecto se basa en una aplicación real y en necesidades reales del sector. No se trata solo de aprobar, sino de adquirir una forma de trabajar profesional.

Fases del proyecto

Fase 1 – Contexto y análisis

Análisis del funcionamiento real de un taller mecánico.

Fase 2 – Diseño de la base de datos

Modelos entidad–relación, normalización y justificación técnica.

Fase 3 – Implementación (SQL)

Creación de tablas, restricciones e inserción de datos.

Fase 4 – Consultas y explotación

Consultas SQL reales y uso desde la aplicación web.

Fase 5 – Integración y cierre

Revisión, documentación y entrega final.

Qué se espera de ti como alumno/a

  • Participación activa
  • Trabajo constante
  • Capacidad de revisión y mejora
  • Justificación técnica

Mensaje final

Este curso no va de memorizar comandos, sino de aprender a razonar, diseñar y construir sistemas reales.

La tecnología cambia, pero la capacidad de razonar y justificar soluciones es lo que te hará un buen profesional.
